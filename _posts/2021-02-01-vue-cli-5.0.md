[Vue CLI 5.0.0](https://github.com/vuejs/vue-cli/releases/tag/v5.0.0) is out!✨

As usual, you can run `vue upgrade` to upgrade your CLI project to this new version.

The last major release was in October 2019.
This new release brings new features and the stable support for Vue 3 😍

Let's dig into the most notable changes!

## Webpack 5

The CLI team worked hard to make the CLI compatible with Webpack v5.
It now uses the new Webpack version.

A new plugin `@vue/cli-plugin-webpack-4` has been introduced under the hood in case you still need to use Webpack v4 (as some Webpack plugins of the JS ecosystem haven't catch up).
To opt out of Webpack v5 and use v4, you can add the plugin with `vue add webpack-4`.

## TypeScript

The TS plugin has been updated to use the latest underlying Webpack plugins.
The latest TS version (currently v4.1) is of course supported,
and the minimal version is now v3.6.

As [TSLint is now deprecated](https://github.com/palantir/tslint/issues/4534),
the TSLint support has been removed.
You are encouraged to migrate to ESLint instead.

## ESlint

As the TSLint support is now deprecated,
ESLint is becoming the recommended option used.
The plugin has been updated to use ESLint v7.
A new option has been added to the plugin, allowing to specify an output file for the ESLint results
with `--output-file`:

    yarn lint --format json --output-file results.json

## Jest

The Jest unit test plugin now uses [Jest v26](https://jestjs.io/blog/2020/05/05/jest-26).
It should be faster than the previous version of the plugin that used v24 (which was 2 years old).

## Cypress

My favorite e2e testing tool can now be used in its latest version in CLI v5.
It used to be a bit painful, as the CLI v4 was depending on Cypress v3 directly (a quite old version).
Cypress is now a peer dependency of the plugin,
so we can specify the version explicitly in our `package.json`,
and use the shiny new Cypress v6.x \o/.
